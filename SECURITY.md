# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x  | :white_check_mark: |

We support the latest minor release. Security fixes may be backported to prior minors on request.

## Reporting a Vulnerability

**Do not** open a public GitHub issue for security vulnerabilities.

1. **Email** the maintainers (or use [GitHub Security Advisories](https://github.com/wijeratne-a/Catenar/security/advisories) if available)
2. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Impact assessment
   - Suggested fix (if any)

## Response Timeline

| Severity  | Initial Response | Fix Target   |
| --------- | ---------------- | ------------ |
| Critical  | 24–48 hours     | 7 days       |
| High      | 1 week          | 30 days      |
| Medium    | 2 weeks         | 90 days      |
| Low       | 1 month         | Best effort  |

We will acknowledge receipt and provide status updates as we investigate. Public disclosure will be coordinated after a fix is available.
